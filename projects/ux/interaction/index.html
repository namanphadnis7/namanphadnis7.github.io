UMD Overwatch App


Objective

To design a mobile application that simplifies communication between members of an esports team.

Project Details

Duration: 2 months (Mar 2022-April 2022)


Role: UX Designer

Challenges

• Maintaining the collegiate athletic style and working with Maryland color schema throughout the app.


• Designing very specific solutions for a niche audience.

Solution

• A single-stop mobile app that brings all the essential esports team information to one place.


• Similar to an office workspace designed specifically for a small team.


Benefit

• Centralizes all the team information.


• Eliminates the need to juggle between multiple gaming and communication applications for group chats, match schedules, player availability, etc.

Overview

This app was designed for a 'small-scale audience' for an academic project. I designed it for UMD's Overwatch Team(8 players and 2 staff members) in Figma using wireframes and high-fidelity prototypes.

Low-Fidelity Prototype

Based on the initial sketches and mockups, I created a low-fidelity prototype in Figma, showing how the app navigation would look after its component design.

Component Library

I created a design system based on Maryland's colors and collegiate athletic-like fonts. Using these, I created components to use throughout my app.

User Persona

I created a user persona based on the common personality traits and patterns of members of the UMD Overwatch team. This helped me understand my target users better and make contextual improvements to the app in iterative cycles.

User Journey

Before finalizing the design, I was asked to showcase the user persona's journey throughout one day also using other mobile apps made by classmates.

Reflections

• I understood the value of the time and effort that design systems and components save, especially in large scaled projects.


• I learned to not get attached to my designs while following established design guidelines, and styles/themes given by the client.


What's Next?

• The prototype can be used as a starting point by UMD Esports if they decide to develop an intra-communication app.


• The concept can also be utilized by other non-esport small-scale teams to simplify their specific work environment.
