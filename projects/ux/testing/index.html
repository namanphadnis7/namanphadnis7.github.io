Usability and Accessibility Testing


I tested the Overwatch League's offcial website for its usability through user-based testing as well as heuristic evaluations. I then compared the data from both tests to find problem trends and came up with solutions to overcome them.


I also performed an accessibility test through an automated accessibility tester and manual inspection.

Objective

To investigate accessibility and usability issues on the website using automated and manual inspections, and create solutions to overcome them.

Role

• Usability Tester

• Accessibility Tester

Challenges

• Coming up with apt user tasks that give the most relevant insight about the lack of usability in the website.


• Creating solutions that solve the existing issues without significantly altering the current state and layout of the website.

Final Result

Insights on the existing problems.

Concept solutions solving said problems.

Usability Testing

User Tasks

After taking their consent and giving them instructions, I gave the testers some tasks with mixed difficulties to do across the website, noting how easy they found it to be and their time performance while doing it.


The users were also encouraged to use their mics to point out anything that stood out to them during the process.

Performance Analysis

All of the data collected from the observations was analyzed together to find common trends like the most difficult task, the longest task, and what was easy to navigate.


I used the TryMyUI app to conduct these tests, and analyzed the auto generated SUS report for these trends.

Heuristic Evaluation

I then used Schneiderman's 8 golden rules of interface design to evaluate the Overwatch League website as an expert.


Using the principles I learnt in my HCI fundamentals class and the golden rules, I listed the problems I found within the website.

Accessibility Testing

Manual Inspection

I inspected the website for violations of the WCAG 2.0 guidelines at A and AA levels.


I used the NVDA screen reader to peruse through the website and came across several A-level errors, and a few AA-level ones.

Automated Accessibility Scan

I used the WAVE extension to do an automated scan for the website's accessibility.


After letting the software scan the website, it displayed all the WCAG violations and contrast errors that it discovered in the results.
